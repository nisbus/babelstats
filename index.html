Welcome to Babelstats.

The Babelstats application is useful for serving data from a database through a restful interface. It has other features too :)

Philosophy

All numerical data can be treated as a time-series (static data will just have one date). All numerical data can be categorized using 6 dimensions (6 degrees of separation).

Getting started

To start using Babelstats you need to have 3 things installed on your machine:

  • Babelstats
  • CouchDB (You can use a hosted solution like cloudant)
  • nginx (if you host both babelstats and couchdb on the same machine) See the nginx.config in the babelstats repository
After you have setup all the necessary things you will need to get the couchapps into couchdb.
There are several ways to do that (couchapp, kanso, erica).

Now that the database is prepped and your server is running you need some documents in the database.
Take a look at the document_creator.erl which creates some sample documents. You might want to change something in there like the categories and titles if you like, or even start with real data.
After your data is in couchdb you should be able to browse to http://localhost/babelstat/index.html and see the interface for charting data from the database.
The GUI is very crude and only meant for demonstration purposes.

Why?

When dealing with a lot of statistical data from different sources, comparisons get quite hairy. Imagine getting the GDP (Gross domestic product) of all the countries in the world into a single database.
All of the numbers are specified in the local currency of the country and the scale of the economy (billions, millions etc.)
How do you compare them on a graph?
Babelstats lets you put the data in to the database as the source specifies it, you only need to let it know a few metrics of each datapoint.

  • Metric: The metric can be a currency, temerature, measurement (km, miles etc)
  • Scale: The scale is a multiplier of one unit of metric
  • Frequency: This can be hourly, daily etc. and is used to aggregate lower frequency data when comparing with higher frequency data.
Example: US GDP: Metric=USD, Scale=1000000,Frequency=monthly.
When you query Babelstats you can give it the metrics you want the result to be and it will convert the data for you before returning.

Another powerful feature are the calculated series.
A lot of statistical information is derived from base statistics i.e. GDP per capita is the Gross Domestic Product divided by the population of a country.
Imagine not having to store all this derived data but calculate it when requested.
A calculated series in Babelstats is a single document that defines the calculation as a string of document queries and mathematical operators.

Example (GDP per capita): {Countries,USA,Economy,Total,Key Statistics,GDP}/{Countries,USA,Census,Total,Population,All}

When you call this series, Babelstats will call the database requesting both of those series and divide one with the other for each series point it finds.
There is no limit as to the recursiveness of the calculations, that is if one of the series in a calculation is a calculation itself it will traverse the database until it gets to the source and then calculate upwards towards the result.
Of course metrics play a key role here in converting all parts of the calculation to a common logical metric.
